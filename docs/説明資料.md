# CareerSupport — 説明資料

リアルタイム音声認識・翻訳デスクトップアプリケーション

---

## 概要

E-learning やプライベートな会議で、話されている内容をその場で **文字起こし・翻訳** するデスクトップアプリです。
処理は **完全ローカル** を基本とし、より高品質な要約・対話が必要な場面のみ、クラウドAI（Gemini）に切り替えて利用できます。

## 制作期間

**2026年6月1日 ～ 2026年6月29日**（約1ヶ月 / 個人開発）

## 制作の目的・意図

- 外国語の講義や会議を、**リアルタイムに理解** したい。
- 社内会議や学習内容は機密性が高く、**音声をクラウドに送信できない** 場面が多い。
- 既存のクラウド型ツールは API 依存のため、**遅延・コスト** が課題だった。

→ そこで、**完全ローカルで動作してプライバシーを守りつつ、必要な時だけクラウドAIも使える** 柔軟なアプリを目指しました。
特に E-learning では、字幕を追う負担を減らして内容理解に集中でき、さらに **文字起こしの保存や AI 要約で復習にも活用** でき、学習効率を高めます。

## 主な機能

- **デュアルSTT**：Whisper（多言語・GPU）と ReazonSpeech（日本語特化・CPU）をワンクリックで切替
- **リアルタイム翻訳**：Qwen 3（ローカルGPU）／ Google翻訳（フォールバック）
- **AIチャット**：ローカル Qwen ⇄ クラウド Gemini をワンクリックで切替（ストリーミング応答）
- **話者分離**（pyannote.audio）：最大5人の話者を識別
- **TTS読み上げ**：翻訳結果を自動音読（フィードバック防止機能つき）
- **Auto 文区切り（VAD）**：発話の切れ目を検出し、文単位で翻訳
- **文字起こし保存**：会議内容を txt / md / srt で記録
- **多言語対応**：日本語・英語・中国語・ベトナム語・ミャンマー語

## 制作の過程で努力した点

1. **TTS フィードバックループの解決（最も苦労した点）**
   システム音声を録る仕組みのため、読み上げた TTS の音まで再キャプチャしてしまい、自分の声を無限に認識する問題が発生。
   「言語フィルタ＋録音継続フラグ＋直近 TTS テキストの重複除去」という **3層の対策** により、録音を止めずに自分の声だけを除外する方式にしました。

2. **リアルタイム性と UI 応答性の両立**
   重い ML 推論を **マルチスレッド＋Queue（FIFO）** で UI スレッドから分離。認識・翻訳・読み上げが互いをブロックせず、UI が常に応答する設計にしました。

3. **ローカル / クラウド AI の統合**
   翻訳・チャットをローカル Qwen で動かしつつ、**同一インターフェースで Gemini に切替可能** に抽象化。API キーは `config.json` に分離し、コードにハードコードしない設計にしました。

4. **Auto 文区切り（VAD）**
   固定秒数のチャンクが文を途中で分断し翻訳品質が落ちる問題を、**無音検出による文末判定** で解決。実際の発話長を計測することで、ノイズによる誤作動も防ぎました。

5. **UI フレームワークの移行（tkinter → PySide6）**
   バックエンドを変更せず UI 層のみを刷新し、QSS テーマや設定ドロワーなどのモダンな UI を実現しました。

## 動作環境・実行方法

1. 依存パッケージをインストール
   ```
   pip install -r requirements.txt
   ```
2. `config.example.json` を `config.json` にコピーし、必要に応じてキーを設定
   （Gemini / HuggingFace のキーは任意。未設定でもローカル機能は動作）
3. アプリを起動
   ```
   python main.py
   ```

> ※ STT・翻訳モデルは初回実行時に自動ダウンロードされます。
> パッケージ・モデルは同梱しておらず、`requirements.txt` から **再インストール可能な状態** です。
